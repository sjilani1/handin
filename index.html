Development Project

Overview

Goal: To create an interactive interface that filters, sorts and aggregates favorites from a dataset

My Design

  • The interface allows users to go through their matches and add favourites to the match-list.
  • The users may filter through age, location or interests of the person
  • The list can be sorted by age (lowest to highest) or by alphabetical order
  • The user has the freedom to remove a match from the match-list or refresh their filter options

    • Comparative Analysis

    I conducted a comparative analysis of three websites: Amazon, Hulu and Instagram

    Hulu

  • Navigating through websites to watch and adding to ‘My Stuff’ for easy access or ‘liking’ it to see similar content
  • Users can save favourite shows to go back to but they will not be organized based on genre etc

    • Amazon

  • Navigating through a shopping portal where users can add to cart or add to a list for later
  • users add items they wish to purchase to a cart and similar content appears

    • Instagram

  • Saving videos in folders through instagram to be able to access the content later
  • This keeps all your content saved in different folders based on a theme that users can go back to and it also boosts the content for the account that created it

    Analysis

    Amazon and Instagram seem to have the highest ease of use and clear straightforward way of bucketing items in certain lists. Both also allow for custom categorisation which gives the user more freedom. Hulu and Amazon go a step further by offering personalisation and showing similar content based on what has been selected. Amazon and Hulu also have more filter options based on genre/category, price, dates, reviews whereas instagram only sorts through date in which the item was added to the saved category.

    Amazon

  • Strength: easiest navigation and the clearest way to form lists based on color and symbolism in the interface design
  • Weakness: overload of information/ similar content which can sometimes be distracting.

    • Hulu

  • Strength:simple interface design - having a nav bar element pointing to ‘My Stuff’ and a simple + button
  • Weakness: not intuitive to organise all favourites in one bucket, they already have previosuly watched list on the home page so this list is redundant

    • Instagram

  • Strength: simple and easy to use button, saved collection stored seperately and easy to customise
  • Weakness: only filters by date and reclicking the save button removes an item from the list

    • My Design Features

  • Simple and very clearly marked button to indicate how to add to a list
  • 2-3 filter options that work closely with sorting the data to give users options without overloading them with unnecessary features
  • Resetting filters and sort buttons
  • Clear organisation of cards as well as a clear organisation when added to a saved category list
  • The same item cannot be added to the list twice - similar to Hulu and Instagram each item in my list is unique so it does not make sense to add it twice
  • A separate add to list and remove from list button because the lists are stored in different areas and sometimes someone may double click by accident which does not mean remove this item from the list
  • I am not adding personalisation in this tool as most of the filter options allow you to screen for interests, age etc and create your own personal sub-list to choose from

    • Skills/Takeaways

  • Creating an interface that clearly depicts buttons in a way that it would be understandable even in a different language
  • Having simple organization and filter options
  • Allowing users to keep changing parts of the page without having to reload the page
